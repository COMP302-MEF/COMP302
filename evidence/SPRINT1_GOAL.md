# Sprint 1 Goal

## Project

InClass Platform

## Sprint

Sprint 1

## Sprint Goal

The goal of Sprint 1 is to build the foundation of the InClass Platform so that instructors and students can log in, the system can connect to PostgreSQL, and basic course/activity data can be listed through the backend and frontend.

## Sprint 1 Focus

Sprint 1 focuses on:

- PostgreSQL database setup.
- Backend project structure.
- User, course, enrollment, and activity models.
- Basic instructor and student login.
- Basic role separation.
- Instructor course listing.
- Student enrolled course listing.
- Initial activity listing.
- Initial frontend pages for login, register, instructor dashboard, and student panel.

## Sprint 1 Success Criteria

Sprint 1 is successful if:

1. PostgreSQL connection works.
2. Demo users can be created.
3. Instructor can log in.
4. Student can log in.
5. Instructor can see assigned courses.
6. Student can see enrolled courses.
7. Activities can be listed from the backend.
8. Frontend can communicate with the backend.
9. Basic project structure is ready for Sprint 2 AI tutoring and scoring features.

## Sprint 1 Demo Scope

At the end of Sprint 1, the team should be able to show:

- Backend running with FastAPI.
- PostgreSQL connection.
- Login/register pages.
- Basic instructor dashboard.
- Basic student page.
- Course and activity listing through API or UI.

## Notes

Sprint 1 does not aim to complete the full AI tutoring and scoring workflow. Those features are planned for Sprint 2.
