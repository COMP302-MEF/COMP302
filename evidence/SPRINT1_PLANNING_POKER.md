## Discussion Notes

- Authentication and authorization were considered high priority because all later features depend on user roles and course access.
- PostgreSQL setup was added as a technical task because database persistence is required by the project.
- Basic frontend pages were included in Sprint 1 to make sure the project can be demonstrated end to end.
- Course and activity listing were kept in Sprint 1 because they are needed before activity management and student tutoring can be tested.
